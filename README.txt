<h2>Source installation information for modders</h2>
<p>This code follows the Minecraft Forge installation methodology. It will apply some small patches to the vanilla MCP source code, giving you and it access to some of the data and functions you need to build a successful mod.</p>
<p>Note also that the patches are built against "un-renamed" MCP source code (aka SRG Names) - this means that you will not be able to read them directly against normal code.</p>

<h2>Setup Process</h2>
<kbd>Step 1</kbd> Open your command-line and browse to the folder where you extracted the zip file.<br><br>
<kbd>Step 2</kbd> You're left with a choice:

<h4>If you prefer to use Eclipse:</h4>
<ol>
  <li>Run the following command: <code>./gradlew genEclipseRuns</code></li>
  <li>Open Eclipse, Import > Existing Gradle Project > Select Folder or run <code>gradlew eclipse</code> to generate the project.</li>
</ol>

<h4>If you prefer to use IntelliJ:</h4>
<ol>
  <li>Open IDEA, and import project.</li>
  <li>Select your <code>build.gradle</code> file and have it import.</li>
  <li>Run the following command: <code>./gradlew genIntellijRuns</code></li>
  <li>Refresh the Gradle Project in IDEA if required.</li>
</ol>

<p>If at any point you are missing libraries in your IDE, or you've run into problems, you can run <code>gradlew --refresh-dependencies</code> to refresh the local cache, or <code>gradlew clean</code> to reset everything (this does not affect your code) and then start the process again.</p>
